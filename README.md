# nx-typescript-7

Benchmark TypeScript 7 (native) against TypeScript 6 in one Nx workspace, across 100 packages.

Based on [jaysoo/nx-ts7](https://github.com/jaysoo/nx-ts7). That repo splits projects between the two compilers with plugin `include`/`exclude` globs. This repo instead registers `@nx/js/typescript` twice over ALL packages, so every project gets both compiler variants as separate targets:

| Target           | Compiler                    | Command                                            |
| ---------------- | --------------------------- | -------------------------------------------------- |
| `build`          | TypeScript 7 (native `tsc`) | `tsc --build tsconfig.lib.json`                    |
| `typecheck`      | TypeScript 7 (native `tsc`) | `tsc --build tsconfig.json --emitDeclarationOnly`  |
| `build-tsc6`     | TypeScript 6 (`tsc6`)       | `tsc6 --build tsconfig.lib.json`                   |
| `typecheck-tsc6` | TypeScript 6 (`tsc6`)       | `tsc6 --build tsconfig.json --emitDeclarationOnly` |

## Setup

TypeScript 7 ships a `tsc` binary but no JavaScript API, so tools (`@nx/js`, `typescript-eslint`, `vite`) still need TypeScript 6:

```json
{
  "devDependencies": {
    "@typescript/native": "npm:typescript@^7.0.2",
    "typescript": "npm:@typescript/typescript6@^6.0.2"
  }
}
```

- `typescript` alias stays on `@typescript/typescript6` so `require('typescript')` returns a compiler API. It provides the `tsc6` binary.
- `@typescript/native` exists only to supply the TS7 `tsc` binary.

After any install that changes TypeScript packages, verify the binary (npm hoisting can let `@typescript/old` steal `.bin/tsc`; a fresh `npm ci` always resolves it):

```shell
./node_modules/.bin/tsc --version   # Version 7.x
./node_modules/.bin/tsc6 --version  # Version 6.x
```

## Workspace

100 packages (`packages/lib001`..`lib100`), generated by `tools/generate-packages.mjs`:

- 10 layers of 10 packages; each package in layer L depends on 2 deterministic packages from layer L-1 (TS project references mirror the graph, `nx sync` clean)
- ~300 lines of type-heavy code per package (discriminated unions, mapped types, generics)

Both compiler variants share `tsconfig.lib.json` and emit to the same `dist/`, so the comparison is same inputs, different compiler. That also means the targets clobber each other's output and tsbuildinfo -- this is a benchmark repo, always clean between variant switches:

```shell
rm -rf packages/*/dist
```

Note: the inferred `typecheck` target depends on `build` (and `typecheck-tsc6` on `build-tsc6`), so the typecheck benchmark includes builds. Symmetric for both compilers.

## Run

```shell
npm install
npx nx run-many -t build           # TS7
npx nx run-many -t build-tsc6      # TS6
npx nx run-many -t typecheck       # TS7
npx nx run-many -t typecheck-tsc6  # TS6
```

## Benchmark

```shell
hyperfine --warmup 1 --runs 3 \
  --prepare 'rm -rf packages/*/dist' \
  -n 'build (TS7 tsc)' 'npx nx run-many -t build --skipNxCache' \
  -n 'build-tsc6 (TS6 tsc6)' 'npx nx run-many -t build-tsc6 --skipNxCache'

hyperfine --warmup 1 --runs 3 \
  --prepare 'rm -rf packages/*/dist' \
  -n 'typecheck (TS7 tsc)' 'npx nx run-many -t typecheck --skipNxCache' \
  -n 'typecheck-tsc6 (TS6 tsc6)' 'npx nx run-many -t typecheck-tsc6 --skipNxCache'
```

`--skipNxCache` so every run compiles; `--prepare` wipes `dist/` so no run is incremental and neither compiler reads the other's tsbuildinfo.

## Results

Apple Silicon (Darwin 25.5.0), Node 24.11.0, nx 23.0.1, TS 7.0.2 vs 6.0.3, warm Nx daemon, clean `dist/` before every run, Nx cache skipped. hyperfine, 5 runs each (typecheck: 3 runs).

### `nx run-many -t build` vs `nx run-many -t build-tsc6`

| Command                 |       Mean [s] | Min [s] | Max [s] |    Relative |
| :---------------------- | -------------: | ------: | ------: | ----------: |
| `build (TS7 tsc)`       |  5.186 ± 0.042 |   5.118 |   5.220 |        1.00 |
| `build-tsc6 (TS6 tsc6)` | 15.471 ± 0.098 |  15.410 |  15.643 | 2.98 ± 0.03 |

### `nx run-many -t typecheck` vs `nx run-many -t typecheck-tsc6`

| Command                     |       Mean [s] | Min [s] | Max [s] |    Relative |
| :-------------------------- | -------------: | ------: | ------: | ----------: |
| `typecheck (TS7 tsc)`       | 10.043 ± 0.234 |   9.810 |  10.278 |        1.00 |
| `typecheck-tsc6 (TS6 tsc6)` | 20.670 ± 0.240 |  20.405 |  20.872 | 2.06 ± 0.05 |

TS7 is ~3x faster on build and ~2x faster on typecheck (typecheck runs 200 tasks: build + typecheck per project, so it carries extra Nx orchestration overhead in both variants). CPU time gap is larger: build 18.4s user (TS7) vs 79.2s user (TS6), ~4.3x.
